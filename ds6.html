Design in practice

Below is the introduction to this design sprint, which will contextualize it for you.

"For this design sprint, we developed a browser extension focused on improving online user experience. Specifically, we chose to develop a web extension that allows users to select a portion of text (a word or short phrase) and right-click the selection to open a Wikipedia page corresponding to the selected text, providing an instance definition and context for the selected term. Alternatively, the user can click the extension icon in the extensions bar, and a pop-up will appear, allowing the user to search Wikipedia right there and ultimately bringing them to a fresh tab with their Wikipedia page of choice pulled up. We chose to build out functionality to search on Wikipedia because it has a fundamentally different nature from Google, the only existing default option. Wikipedia's nature lies in the fact that it is an online encyclopedia, not a search engine. It provides quick access to foundational knowledge, offers neutral overviews of topics ranging from scientific principles to historical events and figures, and enables users to explore interconnected subjects through internal hyperlinksâ€”making it an ideal starting point for deeper research. Ultimately, this extension is focused on making information easily accessible to users in use cases where easy access to Wikipedia might be more beneficial than to Google."
